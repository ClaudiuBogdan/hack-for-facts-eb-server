## Documentation Standards

- Use clear, concise language
- Include code examples with proper TypeScript types
- Follow existing doc patterns in docs/
- Use tables for comparisons and specifications
- Include mermaid diagrams for architecture

## Key Documentation Files

- `docs/ARCHITECTURE.md` - High-level system design
- `docs/CORE-SHELL-ARCHITECTURE.md` - Implementation patterns
- `docs/MODULE-DEPENDENCIES.md` - Import rules
- `AGENTS.md` - AI assistant instructions

## When Documenting

1. Read existing docs to match style
2. Be specific - include file paths and code examples
3. Document the 'why', not just the 'what'
4. Keep docs up to date with code changes
